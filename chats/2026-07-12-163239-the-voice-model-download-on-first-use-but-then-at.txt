  What was wrong

  The error is a stale-deploy race, not a problem with the model download:

  - The service worker precaches the app shell, but assets/vosk-*.js is
  deliberately excluded from the precache (it's ~6 MB; vite.config.ts
  globIgnores). It's only cached at runtime the first time it's fetched.
  - A returning visitor therefore gets the old app shell from the SW precache.
  When they enable voice for the first time, the old shell dynamically imports
  the old vosk chunk hash — but each GitHub Pages deploy deletes the previous
  hashed assets. I confirmed this live: the deployed site currently ships
  vosk-RkwoQNwM.js, while your browser requested vosk-Dloz6gaQ.js, which now
  returns 404.
  - The model download reaches 100% fine because its filename
  (vosk-model-small-en-us-0.15.tar.gz) is version-stable across deploys; only
  the JS chunk hash moved.

  Since you deploy on every push to main, any user who loads the app and enables
  voice for the first time after a subsequent deploy would hit this.

  The fix

  - src/lib/voice/voice.svelte.ts — the vosk import now goes through a new
  loadVoskModule(): if the import fails with a stale-chunk TypeError, it
  persists the voice-enabled flag (so enabling resumes automatically), nudges
  the service worker to update (bounded 3 s wait for activation), and reloads
  the page once. A sessionStorage flag prevents reload loops; if it still fails
  after the reload (e.g. genuinely offline), the HUD shows a readable message
  instead of the raw browser error.
  - src/lib/voice/staleChunk.ts — pure isStaleChunkError() helper matching the
  Chrome/Firefox/Safari wordings of a failed dynamic import, unit-tested in
  src/tests/staleChunk.test.ts.
  - vite.config.ts — comment on the globIgnores entry cross-referencing the
  recovery.

  One note for the affected device right now: a plain page reload already fixes
  it (the new SW has taken over, and the fresh shell points at the live chunk) —
  the code change makes that recovery automatic for everyone after the next
  deploy. I haven't committed anything; say the word if you'd like a commit.
