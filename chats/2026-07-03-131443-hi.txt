     Piano Tutor — Practice Techniques & Improvements Expansion

     Context

     The app today is a mic-driven Svelte 5 web app with scale/chord reference
     libraries, guided lessons (five-finger, Hanon 1, scale routine, cadences,
     basic sight-reading — C/G/F only, hands separate), free-play transcription,
     and a tuner. For a beginner-to-intermediate player it's missing core
     practice modalities: ear training, rhythm work, repertoire, arpeggios,
     hands-together playing, and jazz material. The user chose to add: ear 
     training, rhythm trainer, expanded technique lessons, leveled 
     sight-reading, jazz content, MIDI keyboard support, and a repertoire mode 
     with graded sheet music + MusicXML/MIDI import.

     Plan is 7 milestones ordered by dependency; M1 (MIDI + input hub) unblocks
     everything else. Each milestone ships standalone value.

     Verified starting points (reuse, don't rebuild)

     - src/lib/practice/matcher.ts — StepMatcher (wait-mode, pitch-only) is
     reused unchanged by ear-training echo, songs, and expanded lessons.
     - src/lib/data/lessons/types.ts — LessonStep {midis, fingers, label?} gets
     extended, not replaced.
     - src/lib/audio/noteEvents.ts — NoteEvent {kind, midi, t, confidence, 
     source: 'mono'|'poly'}; t is AudioContext seconds (clock-domain issue,
     fixed in M1).
     - src/lib/data/scaleFingerings.ts already covers all 36 scales — lessons
     just restrict to C/G/F today.
     - src/lib/notation/vexScore.ts renders one stave, quarter notes only;
     SheetMusic.svelte is its only consumer.
     - src/lib/transcribe/quantize.ts snapping logic informs MIDI-import
     quantization; src/lib/audio/playback.ts gates the mic against hearing
     itself.

     ---
     M1 — MIDI input + unified input hub (build first)

     Instant, accurate polyphonic input for anyone with a digital piano; mic
     stays as fallback. Prerequisite for hands-together grading, rhythm
     precision, and repertoire feel.

     - New src/lib/input/midiParse.ts — pure parseMidiMessage(data, tMs): 
     NoteEvent | null (0x90/0x80, velocity-0 = off).
     - New src/lib/input/midi.svelte.ts — Web MIDI access, reactive device list,
     statechange handling; MIDI timeStamp is already performance.now()-domain.
     - New src/lib/input/noteInput.svelte.ts — the hub: onInput(listener),
     reactive activeNotes: Set<number>, activeSource: 
     'midi'|'mic-mono'|'mic-poly'|'none', start(preferred: 'mono'|'poly'). If
     MIDI connected → use it, never start mic; else fall back per hint.
     detectionMode becomes advisory.
     - New src/components/InputPicker.svelte — device dropdown + mic fallback;
     hides MIDI UI when navigator.requestMIDIAccess absent (Safari). Keep
     MicButton.svelte for the Tuner.
     - Modify src/lib/audio/noteEvents.ts: add 'midi' to NoteSource, optional
     velocity?, and tMs: number (wall-clock performance.now()) stamped by all
     emitters — the common clock every timed feature needs.
     - Modify src/components/LessonPlayer.svelte: subscribe via hub; pressed
     from activeNotes; swap MicButton → InputPicker.
     - Tests: src/tests/midiParse.test.ts; extract hub source-selection into a
     pure function and test it.

     M2 — Ear training

     - Route /ear, new src/screens/EarTraining.svelte, nav link in App.svelte.
     - New src/lib/ear/quiz.ts (pure): interval-recognition generator (leveled
     L1 unison/2nds/3rds/5th/octave → L4 all within octave), chord-quality quiz
     (reuses getChord), call-and-response phrase generator (3–6 notes,
     stepwise-biased, five-finger position).
     - New src/components/QuizCard.svelte (multiple-choice by ear: play via
     playback.ts, 4 options, reveal + replay) and
     src/components/EchoPlayer.svelte (play-back grading: InputPicker +
     PianoKeyboard + StepMatcher, deliberately no sheet music). Don't reuse
     LessonPlayer — it's lesson-shaped.
     - Modify src/lib/practice/history.svelte.ts: optional kind?: 
     'lesson'|'ear'|'rhythm'|'song' and score?: {correct, total} on
     PracticeRecord (backward compatible).
     - Tests: src/tests/earQuiz.test.ts.

     M3 — Grand staff + expanded technique lessons

     - Spike first: two-stave, two-system Minuet excerpt behind a dev route —
     grand staff is the biggest notation risk. Fallback: fixed
     measures-per-system with horizontal scroll.
     - Modify src/lib/notation/vexScore.ts: dotted/16th durations,
     timeSignature, and new renderGrandScore(container, {keySignature, 
     timeSignature?, staves: [{clef, events}]}, highlights?) using
     StaveConnector + Formatter.joinVoices for alignment. Keep manual-VexFlow
     style.
     - Modify src/lib/data/lessons/types.ts: LessonStep += hands?: 
     (Hand|null)[], startBeat?, durationBeats? (default 1); LessonSegment.hand: 
     Hand|'both', clef: 'treble'|'bass'|'grand'. scoreFromSteps routes midis to
     staves by hands[i] (else split at midi 60).
     - Modify src/lib/data/lessons/scaleRoutine.ts: all 12 major + 12
     harmonic-minor keys (fingerings exist), one/two-octave variants,
     hands-together segments.
     - New src/lib/data/lessons/arpeggios.ts (triad arpeggios 1–2 octaves,
     hand-authored fingering table like chordFingerings.ts) and brokenChords.ts.
     - Modify src/screens/Practice.svelte: key/method filters (avoid flat-list
     explosion); "best with a MIDI keyboard" hint on hands-together when
     activeSource is mic-poly.
     - Tests: extend fingerings.test.ts (two-octave expansion, arpeggio
     completeness), notation.test.ts (grand stave routing).

     M4 — Rhythm trainer

     - New src/lib/practice/timingGrader.ts (pure): gradeTiming(expected, 
     onsets, bpm, anchorMs, {swingRatio?, toleranceMs?}) → per-hit
     perfect(±40ms)/good(±100ms)/early/late + missed/extra; swingRatio (default
     2/3) transforms off-beat eighths — swing grading for M6 falls out free.
     - Modify src/lib/audio/metronome.ts: onBeat(cb), getAnchorMs() (convert
     Tone time → wall clock), count-in support.
     - Route /rhythm, new src/screens/RhythmTrainer.svelte; new
     src/lib/data/rhythms.ts — leveled patterns (L1 quarters/halves → L4 swing
     eighths, Charleston, comping figures). Flow: count-in → tap N bars (any
     key, pitch ignored; MIDI is the quality path, piano-tap on mic works) →
     grade against grid. Render pattern on fixed pitch via existing renderScore
     + new durations.
     - Clap detection (src/lib/audio/onsetDetector.ts, RMS-flux) is a stretch
     goal — don't block on it.
     - Tests: src/tests/timingGrader.test.ts, pattern-table sanity (bars sum
     exactly).

     M5 — Leveled sight-reading

     - New src/lib/sightread/generator.ts — makeSightReading(level 1–5, seed?):
     L1 = today's behavior; L2 G/D/F positions + skips + half notes; L3 bass
     clef, keys to 2 accidentals; L4 eighths/dotted + optional timing grade via
     gradeTiming; L5 hands together on grand staff (LH held notes under RH
     melody). Keep lessons/sightReading.ts as thin wrapper.
     - New src/lib/practice/progress.svelte.ts — per-activity level/streak
     persistence ("level up after 3 clean reads"); reused by ear-training
     levels.
     - Modify src/screens/Practice.svelte: level picker + "New melody".
     - Tests: src/tests/sightreading.test.ts — range/key conformance, bars sum,
     deterministic under seed.

     M6 — Jazz content

     - Modify src/lib/theory/scales.ts + types: add 'blues', 'dorian',
     'mixolydian', 'major pentatonic' (tonal has all; mode key signature =
     relative major). Modify scaleFingerings.ts: hand-authored fingerings for
     blues + modes in C, F, G, Bb, Eb.
     - Modify src/lib/theory/chords.ts: add m7b5, dim7, 6 qualities; modify
     chordFingerings.ts + ChordsLibrary.svelte (incl. shell-voicing toggle).
     - New src/lib/theory/voicings.ts (pure): shellVoicing(root, quality, 
     'A'|'B') (1-7/1-3 shells, 3-7/7-3 guide tones), iiVIVoicings(key) with
     smooth voice-leading.
     - New src/lib/data/lessons/jazz.ts: ii-V-I drills (LH roots / RH guide
     tones / hands together, cycle of fourths), 12-bar blues comping (C/F/Bb),
     blues-scale routines.
     - Swing rhythm patterns land via rhythms.ts L4 + swingRatio; 7th-quality
     ear quiz falls out of new chord qualities.
     - Tests: extend theory.test.ts; new voicings.test.ts (shell intervals,
     voice-leading ≤2 semitones/voice).

     M7 — Repertoire mode + graded sheet music (biggest, last)

     - Song model — new src/lib/data/songs/types.ts: Song {id, title, composer, 
     grade 1–4, style, keySignature, timeSignature, tempoBpm, swing?, 
     sections[], measures[{notes: SongNote[]}]}, SongNote {midi, startBeat, 
     durationBeats, hand, finger?, tieToNext?}.
     - Pipeline: MusicXML authored/cleaned in MuseScore from public-domain
     sources (≤1930; engrave ourselves, no community-score scraping) →
     content/musicxml/*.musicxml → new scripts/convert-songs.ts (npm run songs,
     via vite-node + jsdom for DOMParser) → committed JSON in
     src/lib/data/songs/generated/.
     - Parser — new src/lib/songs/musicxml.ts (DOMParser-based, shared
     browser/build): strict v1 subset — single piano part, ≤2 staves, 1
     voice/staff, durations to 16ths + dots, ties, chords, key/time/tempo.
     Anything else = loud error. Tuplets/multi-voice/repeats out of scope (write
     out repeats in source).
     - Grading bridge — new src/lib/practice/songSteps.ts: stepsFromSong(song, 
     {hands, fromMeasure, toMeasure}) collapses equal-start notes into chord
     steps → wait-mode is just StepMatcher. Extract shared grading loop from
     LessonPlayer into new src/lib/practice/gradingSession.svelte.ts and
     refactor LessonPlayer onto it.
     - UI — route /songs, new src/screens/Songs.svelte (graded grid + import)
     and src/components/SongPlayer.svelte: multi-system grand staff (new
     src/lib/notation/layout.ts for measure/system packing +
     GrandSheetMusic.svelte; keep SheetMusic.svelte untouched), cursor
     highlight, section looping, RH/LH/both, demo playback via new
     playSong(song, bpm) in playback.ts. Timed mode via gradeTiming is stretch.
     - Import — new src/lib/songs/midiImport.ts (SMF 0/1, quantize to 16th grid
     reusing transcribe/quantize.ts approach), MusicXML import reuses parser;
     new src/lib/songs/userSongs.svelte.ts (localStorage, capped).
     - Starter list (14): G1 — Ode to Joy; Aura Lee; Hot Cross Buns/Mary combo;
     When the Saints (straight + swing toggle). G2 — Petzold Minuet in G BWV
     Anh. 114; Burgmüller Arabesque; Schumann Melody Op. 68/1; Greensleeves;
     house-written 12-bar C blues with shell comping (pairs with M6). G3 —
     Burgmüller La Candeur; Clementi Sonatina Op. 36/1 mvt 1; Bach Prelude in C
     BWV 846; The Entertainer (simplified A strain, own arrangement). G4 —
     Burgmüller Ballade; St. Louis Blues (Handy, 1914).
     - Tests: musicxml.test.ts (fixtures → Song: ties, chords, durations),
     songSteps.test.ts, layout.test.ts, converter validation of all bundled JSON
     (measures sum to time signature, midis 21–108).

     Key risks

     1. VexFlow grand staff/multi-system — mitigate with M3 spike; fallback
     fixed-width systems + scroll.
     2. Clock domains (mic AudioContext vs Tone vs MIDI) — fixed structurally by
     tMs wall-clock stamping (M1) + Tone→wall conversion in metronome (M4);
     ±40/±100ms tolerances absorb jitter.
     3. Mic poly latency (~1s) makes hands-together/rhythm/songs feel broken on
     mic — MIDI is the quality path; UI must say "best with a MIDI keyboard",
     not silently degrade.
     4. Web MIDI is Chromium-only — mic remains universal fallback; hide MIDI UI
     when API absent.
     5. MusicXML scope creep — strict subset, hard errors, pick pieces that fit.

     Verification

     - Per milestone: npm test (Vitest) — every new pure module gets tests in
     the existing src/tests/ style (matcher/theory/quantize precedents).
     - Manual end-to-end per milestone with npm run dev: M1 — connect a MIDI
     keyboard (or use mic fallback) and complete an existing lesson; M2 — take
     an interval quiz and echo a phrase; M3 — play a two-octave B major scale
     hands together from the grand staff; M4 — tap a syncopated pattern and
     check early/late feedback; M5 — level up sight-reading through 3 clean
     reads; M6 — play a ii-V-I drill in F; M7 — run npm run songs, open Minuet
     in G, loop a section in wait-mode, import a MIDI file.
     - Without hardware: mic mono path covers single-line modes; MIDI parse
     logic covered by unit tests.
