  What changed

  The leak: both practice players always rendered the piano keyboard with the
  next expected key lit green plus a finger-number badge, and the sight-reading
  generator wrote finger numbers into every step, which the score renders — in a
  five-finger position, "finger 3" literally names the note. You could complete
  any reading exercise without decoding a single notehead.

  The fix — a new hints: 'technique' | 'reading' field on Lesson
  (src/lib/data/lessons/types.ts):

  - Reading mode (sight-reading, all 5 levels, and all songs): the keyboard is
  hidden by default. A "Show keyboard" checkbox brings it back, but even then it
  only mirrors what you played (plus red wrong-note flashes) — never the
  expected keys or fingers. The on-screen hint now says "read it off the staff"
  instead of "green keys show where it is".
  - Technique mode (everything else — unchanged default): full keyboard with
  expected keys and finger badges.

  Which lessons keep fingering, and why: fingering stays for five-finger 
  positions, Hanon, scale routines, melodic minors, chromatic scale, contrary 
  motion, arpeggios, broken chords, cadences, and jazz voicings — there the
  fingering is the pedagogy (the hand-authored ABRSM/Hanon charts exist
  precisely to drill thumb-under patterns and chord hand-shapes), and printed
  editions of these exercises always show it. Sight-reading keeps exactly one
  fingering: a starting finger on the first note of each exercise (LH gets 5 on
  the tonic in the bass-clef level) — this mirrors real ABRSM sight-reading
  tests, where the opening finger is a hand-position cue and everything after
  must be read. Songs never rendered fingering on the grand staff anyway; their
  per-note finger data only surfaced through the now-removed keyboard badges.

  Tests updated: sightreading.test.ts now asserts every level is hints: 
  'reading' with fingering on the first note only. All 504 tests and npm run 
  check pass.

  One thing to consider later: in reading mode wrong notes now only show as a
  running count (the red flash lived on the keyboard). If that feels too quiet
  in practice, flashing the played wrong note on the staff would be the
  notation-first way to restore it.
