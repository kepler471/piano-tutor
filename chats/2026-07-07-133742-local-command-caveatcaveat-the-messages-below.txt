     Beginner → Intermediate Learning Path

     Context

     The app has strong raw material — 24 scale lessons, Hanon, arpeggios,
     cadences, jazz drills, graded songs (1–7), ear/rhythm/sight-reading levels
     — but no curriculum connecting them. A beginner lands on 8 equal Home tiles
     with no path, no "start here", and no sense of what order to learn things
     in. The goal: make the app able to guide someone from beginner to early
     intermediate (~ABRSM Grade 3–4), with a learning guide that also points to
     external resources the user should supplement with.

     User decisions:
     - Learning guide in both forms: in-app screen + docs/GUIDE.md (single
     source of truth, generated).
     - Fill technique gaps: chromatic scale, contrary motion, melodic minors,
     black-key arpeggios.
     - More repertoire: complete short public-domain pieces for grades 2–5.
     - Theory mini-lessons in the guide, linking to existing libraries.
     - Expand quizzes substantially (user loved the quiz format) — theory
     quizzes + ear/rhythm expansion.
     - External resources: method books, syllabi, YouTube teachers,
     teacher/community advice.
     - NO new tracking: keep existing history/streaks untouched, but the guide
     is a static map — no checkboxes, no completion state, no stage persistence.

     Phase 1 — Router params, guide data, GuideScreen

     Router query params — src/router.svelte.ts

     Currently bare hash strings. Add query support, keeping currentRoute()
     returning path-only so all route === '/x' checks in App.svelte keep

     Record<string, string> }  // pure, tested

     Guide data — src/lib/data/guide.ts (new, pure)

     Single source of truth for the curriculum. Shape:
     type GuideLink =
       | { kind: 'lesson'; lessonId: string; label: string }
       | { kind: 'song'; songId: string; label: string }
       | { kind: 'quiz'; mode: QuizModeId; level: number; label: string }
       | { kind: 'rhythm'; level: 1|2|3|4; label: string }
       | { kind: 'sight'; level: SightLevel; label: string }
       | { kind: 'scale'; root: string; type: ScaleTypeId; label: string }
       | { kind: 'chord'; root: string; quality: ChordQualityId; label: string }
       | { kind: 'route'; route: string; label: string }

     interface GuideStage {
       id: string; title: string; gradeEquivalent: string; overview: string
       goals: string[]
       technique: GuideLink[]; repertoire: GuideLink[]
       ear: GuideLink[]; rhythm: GuideLink[]; sight: GuideLink[]
       theory: { title: string; body: string; links: GuideLink[] }[]
       weeklyPlan: { totalMinutes: number; items: string[] }
       resources: { title: string; url: string; note: string }[]
       moveOnWhen: string[]   // self-assessment prose — NOT tracked
     }
     export const GUIDE_STAGES: GuideStage[]           // 5 stages
     export function guideHref(link: GuideLink): string // → 
     '#/practice?lesson=scale-C-major'

     Five stages:
     1. Foundations (pre-G1): five-finger lessons, Hanon 1, C/G/F majors HS, ear
     intervals L1, rhythm L1, sight L1. Theory: staff, clefs, note names.
     2. ≈ Grade 1: white-key majors HS+HT, C/G/F arpeggios, cadences, grade-1
     songs, ear L2, rhythm L2, sight L2. Theory: key signatures to 1♯/1♭,
     intervals.
     3. ≈ Grade 2: harmonic minors, broken chords, 2-octave scales, grade-2/3
     repertoire, chord-quality ear L2, rhythm L3, sight L3. Theory: relative
     minors, I–IV–V.
     4. ≈ Grade 3: chromatic, contrary motion, melodic minors, black-key
     arpeggios, grade-3/4 repertoire, ear L3 + cadence ID, rhythm L4, sight L4.
     Theory: 7th chords, functions, circle of fifths.
     5. ≈ Grade 4 / early intermediate: all keys 2-octave HT, jazz ii–V–I
     lessons, grade-4/5 repertoire, ear L4 + scale-type ID, sight L5. Theory:
     modes, secondary chords, "when to get a teacher".

     External resources per stage (static https:// links, never fetched): Faber
     Adult Piano Adventures, Alfred's Adult All-in-One, ABRSM & RCM syllabi,
     reputable YouTube channels (Bill Hilton, Piano Lessons on the Web, Josh
     Wright etc.), IMSLP + MuseScore (PD scores), r/pianolearning, teacher
     advice from stage 3.

     Phase 3 — Technique gap lessons (src/lib/data/lessons/)

     All follow the existing scaleRoutine.ts structure (its upDown/handsTogether
     pattern; HT segments use the existing per-segment detectionMode: 'poly'
     override — no detection/player changes needed). Register in index.ts in
     pedagogical order. Fingerings are hand-authored from the ABRSM Manual of
     Scales (cite in docblocks) per project convention.

     - chromaticScale.ts — chromatic from C, D, A; 1 octave HS then HT.
     Hand-authored fingering table (1-3 alternation, 1-2 at the E–F and B–C
     white pairs), keyed by start pitch class.
     - contraryMotion.ts — C, E, F, G major (ABRSM G2–3 staples). Both hands
     same finger numbers simultaneously in same-note contrary motion → build
     from scaleFingerings[...].rh alone; steps {midis:[lh,rh], fingers:[f,f], 
     hands:['L','R']}, poly, grand clef. Test the mirror invariant.
     - melodicMinor.ts — start with A, D, E, G, C. Explicit asymmetric steps:
     ascending = harmonic minor base with raised 6/7, descending = reversed
     natural minor; reuse the existing minor fingering charts. Not added to
     ScaleTypeId/ScalesLibrary (asymmetry breaks the symmetric ScaleInfo/upDown
     model — note in docblock).
     - Black-key arpeggios — extend arpeggios.ts: replace single
     ARPEGGIO_FINGERING with a per-key ARPEGGIO_FINGERINGS table (white-key
     roots keep 1-2-3-5/5-3-2-1); add D♭, E♭, A♭, B♭, F♯ major + B♭, E♭, C♯, F♯,
     G♯ minor. Highest correctness risk — transcribe from one authoritative
     source; tests enforce structure only (valid fingers, no adjacent repeats),
     correctness is human review.

     Phase 4 — Quizzes: unified screen + theory + ear/rhythm expansion

     - Extract round scaffolding from EarTraining.svelte (round-of-10, level
     buttons, "Level up →", existing addRecord call — reusing existing history
     is fine, no new persistence) into src/components/QuizSession.svelte.
     QuizCard.svelte unchanged.
     - Evolve EarTraining.svelte → src/screens/Quizzes.svelte at /quizzes, keep
     /ear as an alias route. Modes grouped: By ear / Reading & theory.
     QuizModeId union in a pure module shared with guide.ts.
     - Theory generators — src/lib/theory/quiz.ts (new, pure, mirrors
     ear/quiz.ts: injectable RNG, LEVELS arrays):
       a. Note naming (4 levels: treble → +bass → ledger lines → accidentals) —
     one-note ScoreModel via existing scoreFromSteps/midiToVexKeyInKey, rendered
     with SheetMusic.svelte.
       b. Key-signature ID (4 levels: 0–1 → 3 → 5 → all 7; minors at top
     levels).
       c. Interval on staff — visual twin of the ear quiz, reuse
     INTERVAL_LEVELS/INTERVAL_LABELS from ear/quiz.ts.
       d. Chord spelling (3 levels: triads → inversions → 7ths) — symbol shown,
     options from getChord(...).noteNames + wrong-quality distractors.
       e. Chord function (text-only: "In G major, which chord is V?").
     - Ear expansion — extend src/lib/ear/quiz.ts: scale-type ID (major/minor →
     +harmonic → +blues/pentatonic → +dorian/mixolydian, reusing SCALE_TYPES)
     and cadence ID (authentic/plagal/half, +deceptive at L3) built from
     getChord.
     - Rhythm dictation (the gap — hearing → notation; tap-along already covers
     reading): play a pattern as clicks, pick the matching notation from 3
     rendered options. Generator pure; new RhythmQuizCard.svelte renders options
     as small SheetMusic scores (QuizCard stays string-only). Add 2–3 patterns
     per level to rhythms.ts for distractor variety.
     - Tests (theoryQuiz.test.ts, extend earQuiz.test.ts): mode × level × ~200
     seeds — options unique, answer present exactly once, answers verifiably
     correct against tonal/getChord.
