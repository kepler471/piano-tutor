     Add a "show fingering" checkbox to the Songs sheet music

     Context

     SongPlayer.svelte already has a "Show keyboard" checkbox that toggles the
     PianoKeyboard
     below the staff. Fingering numbers on the staff itself, however, are always
     rendered whenever
     the underlying note data has a finger value — there's no toggle, and no
     per-render flag to
     suppress them. The user wants a second checkbox, next to "Show keyboard,"
     to show/hide the
     fingering annotations on the music.

     On the sourcing question: I checked. src/lib/data/scaleFingerings.ts
     explicitly documents
     its source ("Standard one-octave scale fingerings (ABRSM / Hanon
     convention)"). The song
     fingerings in src/lib/data/songs/catalog.ts, however, carry no source 
     citation — no
     edition, no ABRSM/Henle/etc. reference — in the code, comments, or commit
     history (Add graded repertoire up to Grade 7…, Add nine graded pieces…).
     They read as the app author's own
     reasonable-but-uncited fingering choices, entered by hand alongside each
     note. I can't confirm
     they trace back to a specific authoritative source, and I don't want to
     claim otherwise. Coverage
     is also partial: 6 of the 18 hand-authored songs (minuet-in-g,
     twelve-bar-blues-c,
     jazz-blues-walking-g, ii-v-i-etude, when-the-saints-stride,
     the-entertainer-stride) have
     zero fingering data, and imported songs (catalogImported.ts, user
     MusicXML/MIDI uploads)
     have none at all — for those, the new checkbox will simply show nothing
     when checked. This plan
     only adds the toggle; it does not touch the fingering data itself. If you
     want the data
     re-verified against a real source (e.g. re-deriving from the Mutopia/IMSLP
     engravings) or the
     uncited status called out to end users, that's a separate follow-up — happy
     to scope it if
     you want.

     Approach

     Thread a showFingering boolean from SongPlayer.svelte down to where
     fingering is actually
     drawn, mirroring the existing showKeyboard checkbox pattern.

     1. src/lib/notation/songScore.ts — songSystems(song, opts): extend the opts
     type
     (currently SongStepOptions & { measuresPerSystem?: number }) with an
     optional
     showFingering?: boolean. At line 98, where each ScoreEvent is built:
     fingerings: opts.showFingering === false ? undefined : group.notes.map((n)
     => n.finger ?? null),
     1. Default (flag omitted) keeps current always-on behavior, so no other
     caller changes.
     2. src/components/GrandSheetMusic.svelte — add showFingering?: boolean to
     the $props()
     destructure (used by both SongPlayer and LessonPlayer... actually only
     SongPlayer uses
     this component; verify at implementation time). Pass it through in both
     songSystems(...)
     calls (the sizing pass at line 36 and the render pass at line 51).
     3. src/components/SongPlayer.svelte:
       - Add let showFingering = $state(true) near showKeyboard (line 27).
     Default true
     since fingering is already shown unconditionally today — the checkbox
     should read as
     "currently on, uncheck to hide," not silently change default behavior.
       - Add a second checkbox next to "Show keyboard" (line 247-249), reusing
     the .show-kb style
     (or a shared class) for visual consistency:
     svelte
     <label class="show-kb">
       <input type="checkbox" bind:checked={showFingering} /> Show fingering
     </label>
       - Pass showFingering into <GrandSheetMusic {song} {range} 
     stepHighlights={highlights} {showFingering} />.

     No changes needed in vexScore.ts — buildNote() already treats an
     absent/undefined
     fingerings array as "draw nothing," so nulling it out upstream is
     sufficient.

     Verification

     - npm run check (svelte-check + tsc) for type correctness across the new
     prop threading.
     - npm run dev, open a song with fingering data (e.g. Ode to Joy), confirm
     the new checkbox
     defaults checked with fingering numbers visible, and unchecking it removes
     them from the
     staff without affecting note rendering, highlighting, or the keyboard
     checkbox.
     - Spot-check a song with no fingering data (e.g. Minuet in G) to confirm
     the checkbox is inert
     there (no visual change either way) rather than erroring.
