     The user hears wrong/clashing notes when playing song demos (▶ Demo in

     flagged Gymnopédie "clashes" are authentic Satie). The audible problems are
     in the playback mechanism (src/lib/audio/playback.ts):

     1. 1 s sampler release tail (release: 1, playback.ts:34) — every note rings
     ~1 s past its end. In chromatic progressions (Chopin prelude ♩=56:
     descending eighth-note chords; walking-bass blues) each chord audibly
     overlaps the next one a semitone away → perceived wrong notes. This is the 
     primary cause.
     2. Swing bug (playSong, playback.ts:128-145) — off-beat eighths get their
     onset moved to beat+2/3 but keep their straight duration: a gap opens at
     x.5→x.667 and the swung note overlaps the next downbeat. Affects the 6
     swing: true jazz/blues songs.
     3. Flat velocity — everything plays at velocity 1.0; melody buried in dense
     textures (Gymnopédie, Chopin).
     4. Tempo review (data, secondary) — tempoBpm is quarter-note BPM by
     convention, so 6/8, 3/8, 2/2 pieces derive slow pulses; a few editorial
     values likely need bumping (by ear).

     Changes

     1. New pure module src/lib/audio/schedule.ts

     No Tone import (project convention: logic pure + vitest-testable). Move
     PlayableNote here, add optional hand?: Hand:

     export interface ScheduledNote { midi: number; timeSec: number; durSec:
     number; velocity: number }

     // Piecewise-linear swing map of the beat axis: within each beat 0→0, 
     0.5→2/3, 1→1
     export function swingMapBeat(beat: number, ratio = 2 / 3): number

     export function schedulePlan(notes: PlayableNote[], bpm: number, swing =
     false): ScheduledNote[]
     export function planEndSec(plan: ScheduledNote[]): number

     Per note: timeSec = map(startBeat) * beatSec, endSec = map(startBeat + 
     durationBeats) * beatSec, durSec = max(0.1, (endSec - timeSec) * 0.95),
     velocity R → 0.9, L → 0.65, absent → 0.85. Applying the same monotonic map
     to start AND end fixes the swing gap/overlap by construction (consecutive
     eighths tile exactly; integer-beat notes are untouched; syncopated quarters
     keep their length). Matches swungBeat in
     src/lib/practice/timingGrader.ts:53 at every fraction the grader handles,
     so playback and grading stay consistent — do not touch the grader.

  What was wrong

  Your ears were right, but the sheet music wasn't the problem. I verified the
  note data two ways: the hand-written arrangements match the real scores
  note-for-note (Für Elise, Minuet in G, Mozart K545, Canon in D, etc.), and the
  Mutopia-imported pieces match a fresh re-import of the original source MIDI —
  even the dissonances my scanner flagged in Gymnopédie turned out to be
  authentic Satie. The wrong/clashing notes came from the playback engine:

  1. A 1-second release tail on the sampler (the main culprit) — every note kept
  ringing a full second after its notated end. In anything chromatic (the
  Chopin prelude's descending chords, the walking-bass blues), each chord
  audibly overlapped the next one a semitone away, which sounds exactly like
  wrong notes.
  2. A swing bug — swung off-beat eighths were delayed to the 2/3 point but kept
  their straight length, so every swung note both left a gap and smeared over
  the following downbeat (all six jazz/blues songs).
  3. Flat velocity — everything played at maximum, burying melodies under
  accompaniment in dense pieces.

  What changed

  - New pure module src/lib/audio/schedule.ts: swing now warps the beat axis for
  both note starts and ends, so eighths tile into clean triplets with no gaps
  or overlaps; right hand plays at 0.9 velocity, left at 0.65. Covered by a new
  src/tests/schedule.test.ts (tiling invariant, swing durations, floor,
  velocity).
  - playback.ts: release dropped from 1 s to 0.3 s (realistic damper), and
  playSong now just maps the scheduled plan onto the sampler.
  - SongPlayer.svelte passes each note's hand through for the velocity shaping.

  All 738 tests pass and npm run check is clean. Two commits: the scheduler +
  tests, then the playback wiring.
