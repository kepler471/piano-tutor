     Sourcing strategy (the part that actually matters)

     Accuracy is the constraint: an LLM must never hallucinate note-by-note data
     for real pieces
     with genuine compositional detail. Split by grade band:

     - Grades 1–2 (hand-authored tuples, no MIDI needed): simple,
     universally-known
     monophonic melodies with a basic LH accompaniment — rounds, nursery rhymes,
     hymns — in the
     same style as the existing twinkle/mary/jingleBells/saints entries in
     catalog.ts.
     Safe to hand-transcribe because these tunes are common knowledge with no
     real ambiguity.
     Do not use this method for anything with real voice-leading or a specific
     notated
     original (e.g. no hand-transcribing a Burgmüller étude "from memory").
     - Grades 2–7 (Mutopia MIDI → scripts/midiToCatalog.ts → hand-curate): the
     same proven
     pipeline already used for catalogImported.ts. Download a real .mid from
     mutopiaproject.org (verified reachable via curl from this environment), run
     npx vite-node scripts/midiToCatalog.ts file.mid to get accurate quantized
     note data, then
     hand-curate: trim/clamp to a teachable excerpt, assign
     id/title/composer/grade/
     style/keySignature/sections. Composers with plenty of untapped Mutopia
     repertoire
     beyond what's already used: more Burgmüller Op. 100,
     Clementi/Kuhlau/Diabelli sonatinas,
     Tchaikovsky Op. 39, Schumann Op. 68, Grieg Lyric Pieces, Bach (Anna
     Magdalena Notebook,
     two-part inventions), Chopin (preludes/waltzes/mazurkas), more Joplin rags,
     Satie
     (Gymnopédies 2/3 + the 6 Gnossiennes).
     - Fingering: only added when the Mutopia source itself encodes it or a
     named public-domain
     edition is cited; otherwise omitted, matching catalogImported.ts's existing
     all-omitted
     convention. Not a blocking deliverable for the 76-song target.
     - Every MIDI-derived file's header comment documents exactly which Mutopia
     pieces were used
     (mirrors catalogImported.ts's existing provenance comment) so the data
     stays auditable.

     validation), execution

     2. Dedup/allocation (single coordinator step) — merge candidate lists,
