     Piano Tutor — General Improvements (4 areas)

     Context

     Survey of the app found it healthy (no TODOs, strong pure-logic test
     coverage, 100+ songs,
     14 quiz modes) but with four clear gaps the user chose to address, all
     confirmed by code
     reading:

     1. Song practice grades note order only — SongPlayer.svelte has no tempo
     control
     (demo always song.tempoBpm), no metronome/count-in, no looping, and never
     calls the
     existing timingGrader.ts even though stepsFromSong already emits startBeat.
     2. Progress is collected but unused — history.svelte.ts read only by Home's
     last-6
     list; progress.svelte.ts levels wired only to sight-reading (quiz/rhythm
     levels reset
     on reload); the guide shows no completion indicators.
     3. Detection — A4=440 hardcoded (noteEvents.ts:27,33, no calibration); in
     mic-poly
     mode grading waits 0.5–1.5 s for Basic Pitch while the ~10 ms mono detector
     is
     display-only (routing.ts forwards poly events exclusively).
     4. Platform — no PWA (not installable/offline despite ~70 MB local models),
     no settings
     screen, preferences don't persist, voice missLog has no UI.

     Four phases, each independently mergeable. Suggested order: 2 → 1 → 3 → 4
     (Phase 3 creates settings.svelte.ts; Phase 4 surfaces it).

     ---
     Phase 1 — Song practice tools

     Files: modify src/components/SongPlayer.svelte,
     src/lib/practice/songSteps.ts;
     create src/lib/practice/songPrefs.svelte.ts; extend
     src/tests/songSteps.test.ts.

     - Tempo slider (40–180, default song.tempoBpm): demo() passes the chosen
     bpm to
     playSong(notes, bpm, song.swing) (already parameterized, verified
     SongPlayer.svelte:117). Persist per song in new songPrefs.svelte.ts
     singleton
     ('piano-tutor.song-prefs', Record<songId, {bpm}>, follow history.svelte.ts
     load/persist pattern).
     - Metronome + count-in: mirror LessonPlayer's wiring
     (LessonPlayer.svelte:99–107) with
     countInBeats: song.timeSignature[0]; restart on attempt reset so each try
     gets a
     count-in. Metronome intentionally does NOT set playback.isPlaying — that
     keeps the mic
     live for grading; do not change.
     - Timing feedback (opt-in, wait-mode stays default): copy LessonPlayer's
     proven pattern
     (verified lines 57–78, 166–175): collect advanceTimes on outcome.advanced;
     on done,
     if metronome on and all steps advanced, gradeTiming(...) anchored to the
     first note
     (advanceTimes[0] - steps[0].startBeat * beatMs), swing-aware. Show "Timing:
     N%" on the
     completion card. Suppress when activeSource === 'mic-poly' (poly latency

     ms-grading meaningless; fused/mono/midi OK).
     - Arbitrary measure range + loop: From/To measure inputs alongside section

     Phase 3 — Detection: A4 calibration + mono/poly fusion

     Files: create src/lib/settings.svelte.ts, src/lib/input/fusion.ts (pure),
     src/tests/fusion.test.ts; modify src/lib/audio/{noteEvents,monoTracker}.ts,

     localStorage

     - A4: add defaulted a4 = 440 params to frequencyToMidi/midiToFrequency
     (all existing callers/tests unchanged); thread through MonoTrackerOptions →
     monoPitch.svelte.ts reads settings.a4 at detection start. Tuner gains an
     auto-calibrate flow: hold A4, average cents for ~1.5 s, newA4 = a4 · 
     2^(avg/1200)

     detection.
     Poly (Basic Pitch) stays at 440 — within ±20 cents its semitone binning is
     unaffected;
     note this in the settings copy and CLAUDE.md.
     - Fusion — new pure MonoPolyFuser in fusion.ts:
     mono on → record onset, forward immediately (instant grading); poly on → if
     an
     unconsumed mono onset with same midi within maxPolyLagMs (default 2000)
     exists, consume
     it and drop (dedup), else forward (chord non-top notes). Exact midi match

     it never blocks progress).
     - Tests: fusion dedup/window/repeat/reset cases + extended chooseSource/
     shouldForward; detection.test.ts synthesizes a 442-based tone → tracker
     with a4: 442
     reads ~0 cents, with 440 reads ≈ +8 cents; calibratedA4 round-trip.
