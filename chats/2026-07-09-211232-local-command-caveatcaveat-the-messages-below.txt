     Smoother missed-notes experience (lookahead/skip)

     Context

     Commit afa3d87 added depth-1 lookahead to StepMatcher: when the mic drops a
     note-on (common on fast playing), playing the next expected note marks the
     current step skipped and continues instead of derailing the run. This
     works well, but the experience is rough around it:

     - Skipped notes render in the same blue as normally-played notes — the
     player can't see what they missed.
     - The completion card says "no wrong notes!" even after 5 misses; skips
     aren't in the progress hint or practice history.
     - Depth is fixed at 1, so two consecutive dropped detections still cause a
     wall of wrong-note flashes.
     - The wrong-note keyboard flash is a fixed 350 ms regardless of tempo, and
     runs containing skips are silently excluded from timing grading with no
     explanation.

     User confirmed all of the above in scope, plus: at the end of a run, show
     which notes were missed (by name, and visibly on the score). EchoPlayer
     stays strict (no lookahead) — it deliberately grades exact reproduction.

     Changes

     1. src/lib/practice/matcher.ts — depth-2 lookahead + missed-note surface 
     (stays pure)

     - Change StepMatcherOptions.lookahead from boolean to 0 | 1 | 2
     (default 0). Only 3 call sites + tests; no compat shim. Update the doc
     comment (currently says "Depth is exactly 1").
     - Replace the single cursor+1 check in onOnset (matcher.ts:68) with a
     shallowest-match-first loop for d = 1..lookahead: on match at depth d, mark
     the d intervening steps 'skipped' (incrementing skips per step), advance
     cursor, clear collected/stumbled, then collect(midi) and return { 
     ...outcome, skipped: d }.
       - Current-step check stays first → repeated notes never skip (preserves
     test at matcher.test.ts:82).
       - Shallowest match wins → a note in both cursor+1 and cursor+2 skips
     only 1.
       - Chord steps: skipping into a chord collects one tone without advancing,
     same as existing depth-1 semantics.
     - Add getter get skippedMidis(): number[] — steps.flatMap((s, i) => 
     results[i] === 'skipped' ? s.midis : []). (Known simplification: a
     partially-collected chord that gets skipped reports all its midis; fine for
     v1.)

     2. src/lib/notation/vexScore.ts — 'missed' highlight + note-name helper

     - HighlightState (line 78): add 'missed'; HIGHLIGHT_COLORS: missed: 
     '#94a3b8' (slate-400 — muted gray, matches the app's slate hint palette).
     buildNote applies fill/stroke generically; no other rendering change.
     GrandSheetMusic.svelte forwards HighlightState opaquely.
     - Add export function midiToNameInKey(midi: number, keySignature: string): 
     string using the existing FLAT_KEYS set (line 97) — FLAT_KEYS.has(key) ? 
     Note.fromMidi(midi) : Note.fromMidiSharps(midi) — and refactor
     midiToVexKeyInKey (line 103) to noteNameToVexKey(midiToNameInKey(...)).

     3. src/lib/practice/history.svelte.ts — record skips

     - Add optional skips?: number to PracticeRecord (optional ⇒ existing
     localStorage records parse unchanged; readers use ?? 0).
     - src/screens/Home.svelte (~line 123, recent-practice line): append · 
     {r.skips} slipped by when (r.skips ?? 0) > 0.

     4. src/components/SongPlayer.svelte

     - Line 42: { lookahead: 2 }.
     - Highlights (line 65): split the collapsed branch — 'corrected' →
     'played', 'skipped' → 'missed'. This shows gray both live and on the
     finished score (the map rebuilds from results each version bump).
     - New deriveds (existing void version pattern): skips = matcher.skips;
     missedNames = deduped matcher.skippedMidis.map(m => midiToNameInKey(m, 
     song.keySignature)), display capped at ~8 names + "…".
     - Completion card (line 207–208): when skips > 0 add a sentence like
     {skips} note(s) slipped by — missed: {missedNames.join(', ')} (gray on the 
     score). Wrong-notes sentence unchanged.
     - Progress hint (line 240–244): add {#if skips > 0}Missed so far: 
     {skips}.{/if}.
     - addRecord (line 88): add skips: matcher.skips.
     - Tempo-relative flash (line 55): replace 350 with flashMs = Math.max(200, 
     Math.min(600, 60000 / song.tempoBpm)) (one beat, clamped). Note: sections
     have no per-section tempi; song.tempoBpm is the only tempo.

     5. src/components/LessonPlayer.svelte

     Mirror SongPlayer: { lookahead: 2 } (line 52), highlight split (line 119),
     skips/missedNames deriveds (spell with lesson.keySignature),
     completion-card sentence (~line 310), progress-hint line (~line 330),
     addRecord skips (line 177), flash using the user-adjustable bpm state (line
     84: Math.max(200, Math.min(600, 60000 / bpm))).

     Plus the timing-grade hint: in the done effect (line 159–188), when the
     metronome/startBeat preconditions hold but advanceTimes.length !== 
     steps.length and matcher.skips > 0, set timingUngraded = true ($state,
     reset alongside timingPct in the effect at line 59). Completion card: {#if 
     timingUngraded} show hint "Timing not graded — some notes slipped by; play
     it again for a rhythm score."

     Deliberately unchanged: oncomplete stays { mistakes } — skips are mostly
     detection artifacts, so they shouldn't block sight-reading auto-advance in
     Practice.svelte. EchoPlayer passes no options → lookahead: 0, unaffected.

     6. src/tests/matcher.test.ts

     - Mechanical: { lookahead: true } → { lookahead: 1 } (lines 44, 55, 61, 68,
     76, 83, 92).
     - Keep the "does not look ahead more than one step" test (line 60) as the
     depth-1 boundary case.
     - New depth-2 describe block: double skip (skipped: 2, results
     ['skipped','skipped','correct',…], skips === 2, mistakes === 0); depth-3
     note is still wrong; shallowest-wins (steps [60],[62],[62], onset 62 skips
     exactly 1); repeated notes never skip at depth 2; double-skip into a chord
     step collects without advancing; double-skip into last step completes;
     skippedMidis contents/order and empty case.

     Verification

     1. npm test — matcher suite + untouched lesson-content tests.
     2. npm run check — svelte-check catches any HighlightState narrowing misses
     and the PracticeRecord change.
     3. Eyeball via the project's verify skill (dev server + Playwright): open a
     song, play note 1, skip note 2, play note 3 → skipped notehead turns gray
     immediately, hint shows "Missed so far: 1"; finish → completion card names
     the missed note and the gray mark persists on the score. Repeat in a lesson
     with metronome on → "Timing not graded…" hint appears. Check Home shows
     "slipped by" on the new record and old history rows still render.

     Risks

     - Depth-2 false skips: a genuinely wrong note equal to the note two steps
     ahead grays out two steps instead of flashing red. Mitigated by
     shallowest-first ordering; accepted for songs and lessons since fast lesson
     material (Hanon, scales) is where detections drop most. Easy to dial back
     per-player later since depth is now an option.
     - Chord steps skipped mid-collection report all their midis as missed
     (slight over-report; acceptable).
